# Changelog

## [0.2.2] - 2026-03-17

### 战斗与数值

- 引入**离散生命值系统**：玩家拥有固定 **2 条命**，受击在防御结算后扣 1 命，生命归零（Life = 0）时立刻结束本局并进入结算。
- 新增**波次间生命恢复**：每当一波战斗清场并进入升级结算时，若当前生命小于 2 条，会自动恢复 **1 命**（不会超过上限），提高长战局的容错率。

### HUD / UI

- 在 HUD 中新增明确的**剩余生命数展示**，与新生命值系统对齐。
- 调整结算 UI，在「生命归零」导致的结束时强调死亡提示（如显示 “You Dead”），让玩家清楚本局是因耗尽生命结束。
- 优化符卡星形按钮 `SpellStarButton` 的大小与位置，并调整 HUD 左上按钮区域布局，提升触控可点性与整体视觉平衡。

### 文档与描述

- 更新 GDD 概览与数值规则章节，详细说明 2 条命生命系统、「生命归零立刻结算」、以及**每波结束恢复 1 命**的规则。
- 更新 README 与主菜单 About 页面，一句话简介同步改为「2 条命离散生命值系统 + 每波结束恢复 1 命」，对外描述与实际玩法一致。

## [0.2.1] - 2026-03-16

### 文档与发布

- 在 GDD 中新增「版本发布信息与检查清单」（第 13 章），统一记录每次发版前需要自检的游戏内/仓库/发布页信息。

## [0.2.0] - 2026-03-16

### HUD / UI

- 重构 HUD 数值展示：主炮与护盾改为左侧 `StatusSlot` 槽位（图标 + 外圈进度环 + `xN` 数量），右侧为副武器与符卡的独立冷却槽条。
- 新增星形符卡按钮 `SpellStarButton`：使用 `star0~star4` 帧序列表现充能进度，冷却中仅显示圆角底框，冷却完成后满星点亮；右下角只在可手动释放时显示，获得自动符卡后隐藏。
- 右侧增加符卡冷却槽：图标改用星形，自动符卡时展示冷却进度，未自动符卡时不显示该槽。
- 删除左侧符卡冷却文本行，仅保留按钮 + 槽位展示，整体布局更简洁。
- 统一使用 `default_ui_theme.tres` 管理字体与样式，删除 HUD 中零散的字体覆盖逻辑。

### 战斗与数值

- 擦弹（Graze）除了加分外，现在也会增加连击值（等同一次命中），对应文档同步更新。
- 提高普通波次出怪量：每波基础敌人数从 4 提升到 6，每波递增从 +2 提高到 +3，使整体刷怪更密集。

## [0.1.1] - 2026-03-13

### 战斗与数值

- **主炮攻速**：实际出手 **75 发/秒** 封顶；**理论射速**可继续随连击/速射升高，超出部分按 **每 1% → +0.02** 折算进主炮伤害（HUD **主炮伤害 ×× [+××]**）。
- **连击攻速**：10 / 25 / 50 / 100 连分档；**100 连后每再多 100 连 +5% 攻速**（倍率 +0.05，线性）。
- **Boss**：主线与续战 Boss **血量加厚**，避免高强化下过快蒸发。

### 续战与流程

- **续战 8 波**，第 **8 波为 Boss**；续关 **+1 稳态护盾**；进入续战前 **连续 3 次三选一**。
- 调试：**跳 Boss** 适配续战；局内设置 **连击 +10/50/100/500 与清零**。

### 武器与符卡

- **炸弹**为副武器：`PlayerBomb`、爆炸 **AoE**、爆炸帧清区域内 **敌弹**、飞行中可 **撞毁敌弹**；与玩家碰撞忽略。
- **符卡**：`spell_*` 与炸弹区分；符卡弹贴图与朝向；**逐发消弹**；可选 **自动符卡** 升级。
- **弓箭**：消弹与高伤；**回旋镖**：回一枚射一枚、词条与逻辑调整。
- 移除 **boss_hunter** 词条。

### HUD / UI

- 左侧 **LeftStatsVBox**：护盾层数、主炮（伤害含溢出、射速仅超上限时显示封顶、齐射分行）、副武器、符卡。
- **Combo** 置于波次旁；DPS / 分数布局调整；关于页展示 **版本号**（与 `config/version` 一致）。

### 修复与平台

- HUD **实际射速**不再因 `0.02` 下限误显示为 50/s；**player.gd** 子弹伤害缩进等修复。
- **擦弹**粒子 Android 可见与层级；**符卡区**与拖动不冲突（短按发动）。
