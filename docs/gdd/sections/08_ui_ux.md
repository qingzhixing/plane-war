# 08 UI/UX（MVP）

## 菜单

- **主菜单场景**：`res://scenes/MainMenu.tscn`（工程 **启动场景**）。进入游戏先见标题与三个入口。**布局在场景内预设子节点**（背景、标题、按钮等），脚本只做信号与 `change_scene`，避免运行时拼 UI。**唯一名称只给必要节点**（如三个入口按钮、`SettingsUI`）；纯装饰/布局节点不设，避免 `%` 名泛滥。
- **主菜单入口**：
  - **开始游戏**：`change_scene` 进入 `res://scenes/Main.tscn` 开战。
  - **设置**：打开与局内相同的设置面板（`show_settings_from_menu()`，不关暂停，因主菜单未跑战斗）。
  - **退出**：`get_tree().quit()`（桌面；移动端无效果可接受）。
- **成绩查询**：主菜单入口「成绩查询」打开全屏遮罩面板，只读展示本地 **`user://records.cfg`** 与局内写入一致的字段：**历史最高得分**、**历史最高连击**、**历史最高 DPS**；无记录时显示 0；关闭返回主菜单。
- **关于（About）**：主菜单「关于」打开说明页：项目名称与简介、开发者、技术栈（Godot 4 等）、**GitHub 仓库链接**（按钮调系统浏览器打开）；文案在场景内预设，便于改链接与署名。
- **局内返回主菜单**：结算界面提供「主菜单」按钮，切回 `MainMenu.tscn`（与「重新开始」区分）。
- 设置：
  - BGM/SFX 音量滑条（0–100%，默认 80%/100%）
  - BGM/SFX 静音开关
  - 震动开关
  - 画面缩放/适配（仅预留入口，可后续实现）
  - **局内提前结算入口**：在设置面板中提供一个“提前结算”按钮，用于当前局直接结束并进入结算界面（等价于 HUD 中的提前结算功能）。

### 设置界面形态（MVP）

- 形式：全屏半透明遮罩 + 居中面板
- 入口：
  - 主菜单中的“设置”按钮
  - 局内可从暂停状态进入（例如在暂停界面增加“设置”按钮）
- 操作规则：
  - 拖动 BGM/SFX 滑条与静音时，实时调用全局 `AudioManager`；**音量与静音持久化到 `user://settings.cfg`（`audio` 段）**，主菜单与局内打开设置前均从 `AudioManager` 同步 UI，避免两套界面状态不一致。
  - 切换静音开关时，仅修改对应通道的静音状态，不改变滑条的数值。
  - **从局内打开设置时**：记录当前暂停状态，自动将游戏 `get_tree().paused` 设为 `true`；关闭设置时恢复为打开前的暂停/继续状态，避免与暂停按钮产生冲突。
  - **从主菜单打开设置时**：不显示局内/调试项（提前结算、跳 Boss、自选升级）；这些仅在 `Main` 对局中从暂停进入设置时可见。

## 局内 HUD

- 局内不显示玩家 HP（已取消玩家生命值系统，玩家不会死亡）。
- 波次（`WaveLabel`，如「第 1 波」；当 Boss 登场且战斗进行中时，可将该区域改为「Boss 战」或在波次后附加标记，如「第 4 波 - Boss」，以强调当前阶段）
- 局内不显示经验条；升级由**波次结束**直接触发，每波清场后默认弹出三选一升级界面。
- 左上角按钮区域：使用一个垂直容器（例如 `TopLeftButtons: VBoxContainer`）承载以下按钮，统一对齐与间距：
  - 暂停按钮（`PauseButton`，左上，**大号触控按钮**，文字“暂停 / 继续”）
  - 设置按钮（`SettingsButton`，用于打开设置界面；**局内“提前结算”仅通过设置面板触发**）
- 符卡/炸弹按钮（若启用，可放在独立容器中）；触发时应有明显的全屏闪光与文字提示，并能感知到“连续爆发”节奏
- **评分与表现相关 HUD（Score / Combo / DPS）**：通过容器组件（如 `VBoxContainer`/`HBoxContainer`）进行分组和对齐，避免单纯靠坐标硬排。
  - `ScoreLabel`：当前分数（Score），推荐放在右上角信息区域的一行，文本形式如「Score: 12345」。
  - `ComboLabel`：单独放在 **`CanvasLayer.layer = -128`**（`ComboBackLayer`），画在 **所有机体/子弹 Sprite 之下**；其余 HUD / 结算 / 升级等仍为默认层，叠在游戏与 Combo 之上；连击提示与边缘 VFX 仍在 HUD 层。  
    - **位置调整约定**：允许在场景中单独摆放 `ComboLabel`（不一定强制在 `TopRightVBox` 内），`hud.gd` 会优先在右上信息容器中查找，若未找到则回退到 `Root` 下查找同名节点。
    - **颜色反馈约定**：连击数越高，`ComboLabel` 的颜色越接近暖色/高亮色，以体现紧张和成就感，例如：
      - 低连击（如 1–9 连）：使用默认/中性颜色；
      - 中连击（如 10–49 连）：使用偏亮的颜色（如黄色）；
      - 高连击（如 50–99 连）：使用更醒目的颜色（如橙色/红色）；
      - 极高连击（如 100 连及以上）：使用**彩虹/循环变色**效果，持续在暖色系之间过渡，强化“特别成就”感。
    - **动态缩放约定**：在 `combo` 每次增加时，`ComboLabel` 会产生一次极短的放大→回弹效果（例如放大到 1.2 倍再还原），不影响可读性但能强调“连击 +1”瞬间。
    - **全屏连击光效约定**：
      - 10 连以上：屏幕四周出现低强度流光边框，提示进入连击状态；
      - 25/50 连：边框流光亮度与流动速度逐档提升，强化“越打越热”的视觉反馈；
      - 100 连及以上：进入全屏炫彩状态（边缘彩虹流光 + 轻度全屏色彩流动叠层），并保持可读性，不遮挡弹幕判定。
  - `DpsLabel`：当前 DPS 与当局最高 DPS，放在 Score 下方，以较小字号显示「DPS: 123  Max: 456」。
  - **`StatsLabel`（属性条）**：与 DPS 同区，多行小字**实时**展示局内数值，便于对照强化与手感；建议包含但不限于：
    - 主武器：**射速（发/秒）**、**齐射弹数**、主炮间隔倒计时（可选）
    - 副武器：**弓箭 / 炸弹** 的冷却剩余与周期（未解锁则显示「—」）
    - **回旋镖**：场上枚数、齐射数（无固定 CD 时仍展示状态）
    - **符卡**：冷却剩余 / 总冷却（与 `Main` 一致）
    - 主武器模式（机炮/弓箭）等与战斗直接相关的只读属性
  - 布局：`DpsLabel` 下方按 **标题 `Label` + 内容 `RichTextLabel`** 分组（与 Score/DPS 同属 `TopRightVBox`）：
    - **MainGunTitleLabel**「主炮」→ **MainGunStatsLabel**（射速、齐射、间隔、**弹速基数与连击弹速 Buff**）
    - **SideWeaponTitleLabel**「副武器」→ **SideWeaponStatsLabel**（弓箭、炸弹、回旋镖）
    - **SpellTitleLabel**「符卡」→ **SpellStatsLabel**（冷却）
  - 各块 `mouse_filter = IGNORE`；内容区 `bbcode_enabled`、`fit_content`、`custom_minimum_size` 保证可见。
> 实现约定：`hud.gd` 以 **`%MainGunStatsLabel`** / **`%SideWeaponStatsLabel`** / **`%SpellStatsLabel`** 引用内容区；标题仅场景摆字。

### Boss HUD

- 局内存在 Boss 时，在屏幕上方单独显示一条 Boss 名称与血条 UI（例如名称在左，血条在右侧横向铺开）。
- 当 Boss 当前 HP > 0 时，该 Boss 血条保持可见，并根据 Boss 剩余 HP 比例实时更新。
  - 当 Boss 被击破（HP 降至 0）后，立即隐藏 Boss 血条 UI；若玩家选择 **继续挑战**，随后回到波次 HUD；若选择 **结算**，再进结算页。
- **Boss 击破后二选一**：**本局结算** / **继续挑战**（威胁+1、一次升级、**续战 4 波**越来越难，结束后结算）；暂停直至点选。

### 连击与评分反馈 UI（新增）

- 连击提示：
  - 当 `combo` > 0 时，在 HUD 上仅显示当前连击数，如「Combo: 12」。连击带来的射速等仍体现在 **主炮** 行（如射速旁 `+xx%`）；主炮伤害为 `基础[+连击]`。**不设连击Buff 独立栏目**。
  - 在连击达到预设档位（10 / 25 / 50 / 100 连）时，短暂在玩家附近或屏幕中央浮现“10 Combo!”、“25 Combo!!” 等文字，并配合轻微缩放/闪光效果。
- 连击中断反馈：
  - 玩家受击或长时间未命中导致连击清零时，短暂显示“Combo Break” 或“连击中断”字样，并播放轻微“失误”提示音（不应过于刺耳，以免挫败感过强）。
- 高分反馈：
  - 当本局分数超过历史最高分或接近某个高分阈值时，可在屏幕一角显示“New Record?” 提示，鼓励玩家继续坚持到关卡结束。

## 升级界面

- 3 张卡片：图标 + 名称 + 一句话效果
- 点击选择后立即回到战斗

## 关卡结束、失败与继续（评分制下的表现方式）

- 游戏的**主反馈从“死亡/失败”调整为“评分/表现”**：  
  - 玩家在局内可以多次受击，受击只会导致连击数清零、评分加成降低、结算评价偏低，不会被强制 Game Over。
- 关卡结束逻辑：
  - 当达到预设关卡结束条件（如 Boss 击破或时间/波次数耗尽）时，清屏并进入结算界面。
  - 结算界面显示：  
    - 标题文案应偏向“结算/总结”而非“死亡/失败”，例如使用「Battle Summary」或「结算」等中性文案，而不是 “You Dead!”。  
    - 本局总分  
    - 最高连击  
    - 当前 DPS 与本局最高 DPS  
    - 历史最高分 / 历史最高 DPS / 历史最高连击（若有记录）
    - 当本局刷新任意一项历史纪录时，在对应行以高亮颜色（如金色）显示，并在括号中展示「当前值（历史最佳）」的对比格式，例如：`Score 12345(12000)`。
- 继续游玩与重新开始：
  - 无玩家死亡界面；仅通过 Boss 击破或提前结算进入结算界面，鼓励玩家完成整局后比拼得分。  
  - 结算界面提供“大号重新开始按钮”，允许玩家立刻再来一局刷更高分。  
  - 结算界面提供“大号回主菜单按钮”（`MainMenuButton`），用于返回主菜单。
  - 若中途需要提前结算，仅通过设置面板中的“提前结算”入口触发，HUD 不再放置单独的提前结算按钮。
