# 08 UI/UX（MVP）

## 菜单

- 主菜单：开始 / 设置 / 退出
- 设置：
  - BGM/SFX 音量滑条（0–100%，默认 80%/100%）
  - BGM/SFX 静音开关
  - 震动开关
  - 画面缩放/适配（仅预留入口，可后续实现）
  - **局内提前结算入口**：在设置面板中提供一个“提前结算”按钮，用于当前局直接结束并进入结算界面（等价于 HUD 中的提前结算功能）。

### 设置界面形态（MVP）

- 形式：全屏半透明遮罩 + 居中面板
- 入口：
  - 主菜单中的“设置”按钮
  - 局内可从暂停状态进入（例如在暂停界面增加“设置”按钮）
- 操作规则：
  - 拖动 BGM/SFX 滑条时，实时调用全局 `AudioManager`，更新线性音量（不打断当前 BGM）。
  - 切换静音开关时，仅修改对应通道的静音状态，不改变滑条的数值。
  - **从局内打开设置时**：记录当前暂停状态，自动将游戏 `get_tree().paused` 设为 `true`；关闭设置时恢复为打开前的暂停/继续状态，避免与暂停按钮产生冲突。

## 局内 HUD

- 局内不显示玩家 HP（已取消玩家生命值系统，玩家不会死亡）。
- 波次（`WaveLabel`，如「第 1 波」；当 Boss 登场且战斗进行中时，可将该区域改为「Boss 战」或在波次后附加标记，如「第 4 波 - Boss」，以强调当前阶段）
- 经验条（`ExpBar`）：固定在屏幕底部靠中位置（宽度约占屏幕 80–90%），用于显示当前经验进度与升级节奏。
- 左上角按钮区域：使用一个垂直容器（例如 `TopLeftButtons: VBoxContainer`）承载以下按钮，统一对齐与间距：
  - 暂停按钮（`PauseButton`，左上，**大号触控按钮**，文字“暂停 / 继续”）
  - 设置按钮（`SettingsButton`，用于打开设置界面；**局内“提前结算”仅通过设置面板触发**）
- 符卡/炸弹按钮（若启用，可放在独立容器中）；触发时应有明显的全屏闪光与文字提示，并能感知到“连续爆发”节奏
- **评分与表现相关 HUD（Score / Combo / DPS）**：通过容器组件（如 `VBoxContainer`/`HBoxContainer`）进行分组和对齐，避免单纯靠坐标硬排。
  - `ScoreLabel`：当前分数（Score），推荐放在右上角信息区域的一行，文本形式如「Score: 12345」。
  - `ComboLabel`：当前连击数（Combo），默认推荐放在 Score 下方；当 `combo > 0` 时显示「Combo: 12」，无连击时留空；连击达到阈值（如 10 / 25 / 50 / 100 连）时在附近短暂弹出“10 Combo!” 等提示。  
    - **位置调整约定**：允许在场景中单独摆放 `ComboLabel`（不一定强制在 `TopRightVBox` 内），`hud.gd` 会优先在右上信息容器中查找，若未找到则回退到 `Root` 下查找同名节点。
    - **颜色反馈约定**：连击数越高，`ComboLabel` 的颜色越接近暖色/高亮色，以体现紧张和成就感，例如：
      - 低连击（如 1–9 连）：使用默认/中性颜色；
      - 中连击（如 10–49 连）：使用偏亮的颜色（如黄色）；
      - 高连击（如 50–99 连）：使用更醒目的颜色（如橙色/红色）；
      - 极高连击（如 100 连及以上）：使用**彩虹/循环变色**效果，持续在暖色系之间过渡，强化“特别成就”感。
    - **动态缩放约定**：在 `combo` 每次增加时，`ComboLabel` 会产生一次极短的放大→回弹效果（例如放大到 1.2 倍再还原），不影响可读性但能强调“连击 +1”瞬间。
    - **全屏连击光效约定**：
      - 10 连以上：屏幕四周出现低强度流光边框，提示进入连击状态；
      - 25/50 连：边框流光亮度与流动速度逐档提升，强化“越打越热”的视觉反馈；
      - 100 连及以上：进入全屏炫彩状态（边缘彩虹流光 + 轻度全屏色彩流动叠层），并保持可读性，不遮挡弹幕判定。
  - `DpsLabel`：当前 DPS 与当局最高 DPS，放在 Combo 下方，以较小字号显示「DPS: 123  Max: 456」，避免喧宾夺主。
> 实现约定：`hud.gd` 只会从场景中查找上述节点名（可在容器内部），不再动态创建 Label；若节点不存在，则对应功能不显示。UI 场景的节点命名需与此约定保持一致，并优先使用 Godot 自带的 `VBoxContainer` / `HBoxContainer` / `MarginContainer` 等组件完成布局。

### Boss HUD

- 局内存在 Boss 时，在屏幕上方单独显示一条 Boss 名称与血条 UI（例如名称在左，血条在右侧横向铺开）。
- 当 Boss 当前 HP > 0 时，该 Boss 血条保持可见，并根据 Boss 剩余 HP 比例实时更新。
- 当 Boss 被击破（HP 降至 0）后，立即隐藏 Boss 血条 UI，避免与后续波次或结算界面产生视觉冲突。

### 连击与评分反馈 UI（新增）

- 连击提示：
  - 当 `combo` > 0 时，在 HUD 上显示当前连击数，如「Combo: 12」。
  - 在连击达到预设档位（10 / 25 / 50 / 100 连）时，短暂在玩家附近或屏幕中央浮现“10 Combo!”、“25 Combo!!” 等文字，并配合轻微缩放/闪光效果。
- 连击中断反馈：
  - 玩家受击或长时间未命中导致连击清零时，短暂显示“Combo Break” 或“连击中断”字样，并播放轻微“失误”提示音（不应过于刺耳，以免挫败感过强）。
- 高分反馈：
  - 当本局分数超过历史最高分或接近某个高分阈值时，可在屏幕一角显示“New Record?” 提示，鼓励玩家继续坚持到关卡结束。

## 升级界面

- 3 张卡片：图标 + 名称 + 一句话效果
- 点击选择后立即回到战斗

## 关卡结束、失败与继续（评分制下的表现方式）

- 游戏的**主反馈从“死亡/失败”调整为“评分/表现”**：  
  - 玩家在局内可以多次受击，受击只会导致连击数清零、评分加成降低、结算评价偏低，不会被强制 Game Over。
- 关卡结束逻辑：
  - 当达到预设关卡结束条件（如 Boss 击破或时间/波次数耗尽）时，清屏并进入结算界面。
  - 结算界面显示：  
    - 本局总分  
    - 最高连击  
    - 当前 DPS 与本局最高 DPS  
    - 历史最高分 / 历史最高 DPS（若有记录）
- 继续游玩与重新开始：
  - 无玩家死亡界面；仅通过 Boss 击破或提前结算进入结算界面，鼓励玩家完成整局后比拼得分。  
  - 结算界面提供“大号重新开始按钮”，允许玩家立刻再来一局刷更高分。  
  - 结算界面提供“大号回主菜单按钮”（`MainMenuButton`），用于返回主菜单。
  - 若中途需要提前结算，仅通过设置面板中的“提前结算”入口触发，HUD 不再放置单独的提前结算按钮。
